## Code Style and Conventions

- Keep documentation up to date as new models or configurations are added.
- Use clear, descriptive names for model configuration files.
- Prefer standard formats (JSON, YAML, TOML) for configuration files.

## Important Notes

- Do not commit large model weight files directly to this repository. Use Git LFS or external storage and reference them instead.
- Keep secrets (API keys, tokens) out of version control.
